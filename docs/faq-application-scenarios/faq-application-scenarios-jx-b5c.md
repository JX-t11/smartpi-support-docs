### 音箱播放音乐时无法识别语音怎么办？

**问题描述：**

使用语音模块开发音箱产品时，遇到播放音乐时无法识别语音指令的问题：
- 功放声音较大（如3W）时，播放音乐过程中语音唤醒无反应
- 不播放音乐或调低音量后，语音识别恢复正常
- 询问是否有支持矩阵麦克风的方案

**原因分析：**

这是典型的**回声干扰**问题：
- 设备自身播放的声音会被麦克风拾取
- 播放音量越大，对语音识别的干扰越严重
- 不支持AEC（声学回声消除）功能的模块无法有效抑制回声

**解决方案：使用 JX-B5C 模块**

JX-B5C 是二合一模组，专门针对蓝牙音箱场景设计：

| 功能特性 | 说明 |
|---------|------|
| 离线语音识别 | 基于CI1302芯片，支持本地语音唤醒和命令词识别 |
| 蓝牙音乐播放 | 集成蜂鸟B芯片，支持A2DP蓝牙音乐播放 |
| AEC回声消除 | 支持声学回声消除，播放音乐时可识别语音 |
| 语音打断 | 播放音乐时可以通过唤醒词打断并识别新指令 |

**应用优势：**

1. **真正的二合一集成**：单模组同时支持语音和蓝牙音乐，无需额外模块
2. **支持打断功能**：播放音乐时说出唤醒词即可打断并识别新指令
3. **高性价比**：相比使用多个模块组合，成本更优

**注意事项：**

- JX-B5C2 IO引脚较少（仅TX/RX），如需连接物理按键可考虑SU-63T
- 使用AEC功能时需按照规格书正确连接麦克风和扬声器
- 详细文档参考：[JX-B5C 官方文档](https://help.aimachip.com/docs/offline_jxb5c)
