# 知识库文档创建与格式要求

## 概述

知识库文档用于系统化管理和分享技术知识，通过标准化的Markdown格式创建结构化的文档内容。

## 文档创建流程

### 1. 上传界面

知识库创建第一步：上传.md文件
- 支持标准Markdown格式文件
- 文件名建议使用英文，避免特殊字符
- 文件大小：建议不超过10MB

![知识库上传界面](/docs/offline-voice-su03t/assets/kb-upload-interface.jpg)

### 2. 设备配置

通过设备行为自定义界面配置知识库内容：

- 设备名称：明确标识设备类型
- 标识ID：唯一标识符
- 功能设置：根据实际需求配置

![设备配置界面](/docs/offline-voice-su03t/assets/kb-device-config.jpg)

## 格式要求

### 文件格式
- **文件类型**：.md（Markdown）
- **编码格式**：UTF-8
- **内容格式**：JSON格式的结构化数据

### 基本结构

```json
{
  "title": "文档标题",
  "description": "文档描述",
  "category": "分类标签",
  "content": "文档内容（支持Markdown）",
  "tags": ["标签1", "标签2"],
  "version": "版本号",
  "author": "作者",
  "created_date": "创建日期"
}
```

### Markdown格式示例

参考提供的"知识库入门示例demo.md"文件，了解具体的格式要求。

## 内容组织

### 1. 文档结构
```markdown
# 主标题

## 概述
简要说明文档目的和适用范围

## 功能特点
列出主要功能点

## 使用方法
详细的操作步骤

## 注意事项
重要提醒和限制条件

## 常见问题
FAQ内容

## 附录
相关参考资料
```

### 2. 图片处理
- 图片路径：使用相对路径
- 图片格式：推荐PNG、JPG
- 图片大小：建议小于2MB
- 命名规范：使用有意义的英文名称

## 编写规范

### 标题规范
- 使用层级标题（H1-H6）
- 标题简洁明了
- 避免使用特殊字符

### 内容要求
- 信息准确、完整
- 语言简洁、专业
- 逻辑清晰、结构合理

### 代码示例
```c
// 代码示例应包含必要注释
void example_function() {
    // 功能说明
    // 实现细节
}
```

## 最佳实践

1. **文档命名**
    - 使用有意义的文件名
    - 避免使用中文和特殊字符
    - 建议使用小写字母和下划线

2. **版本管理**
    - 每次更新修改版本号
    - 记录更新日志
    - 保留历史版本

3. **分类管理**
    - 合理设置分类标签
    - 便于搜索和检索
    - 保持分类体系一致性

4. **质量保证**
    - 发布前进行内容审查
    - 测试代码示例
    - 验证链接有效性

## 注意事项

1. 不要在文档中包含敏感信息
2. 遵守公司文档管理规定
3. 定期更新维护文档内容
4. 保持与其他文档的一致性
