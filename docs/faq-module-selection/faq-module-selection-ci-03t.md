### CI-03T的TTS功能如何选型和喇叭功率如何选择？

**问题描述：**

需要选择支持TTS功能的CI-03T模块，并了解如何选择合适的喇叭功率以满足音量需求。

**解决方案：**

**TTS功能选型：**

1. **CI-03T标准版**：不支持TTS功能，仅支持预录音频播放
2. **CI-03T（TTS版本）**：支持中英文TTS功能
    - 购买时必须选择标注了"文本转语音"的型号
    - 出货时由厂商授权TTS功能
    - 需要烧录专用的TTS固件

**TTS功能使用说明：**

- 支持中英文文本输入
- 可通过串口发送文本进行实时转换
- 支持调节音量、语速、音调参数
- ASR和TTS功能不能在同一模块上同时使用

**喇叭功率选型：**

| 阻抗 | 推荐功率 | 说明 |
|------|----------|------|
| 4Ω | 2.4W起 | 不要超过太大，如4Ω 5W模块带不动 |
| 8Ω | 1.6W起 | 可选8Ω 2W圆喇叭或8Ω 5W腔体喇叭 |

**推荐喇叭型号：**

- **圆喇叭**：8Ω 2W，适合一般音量需求
- **腔体喇叭**：8Ω 5W，适合较大音量需求
- 避免使用**4Ω 5W喇叭**：模块功率可能不足

**应用场景对比：**

| 需求 | 推荐型号 | 说明 |
|------|----------|------|
| 仅离线识别 | CI-03T标准版 | 成本最低 |
| 需要TTS播报 | CI-03T(TTS版) | 支持动态文本 |
| 大音量需求 | 配合功放电路 | 外接功放驱动 |

**注意事项：**

- TTS版本需要额外授权费用
- 购买时确认是否已授权TTS功能
- 大功率喇叭建议外接功放，不要直接使用模块内置功放
- 测试确认音量是否满足产品需求

### CI-03T是否支持离线语音和自学习功能？

需要确认CI-03T模块是否为离线语音模块，以及是否支持用户自学习录音功能，以满足不联网场景下的应用需求。

- 无需联网即可正常工作

**2. 自学习功能支持**

- CI-03T支持自学习功能

**3. 功能特点对比**

| 功能 | 支持情况 | 说明 |
|------|----------|------|
| 离线识别 | ✓ 支持 | 无需联网，本地处理 |

| 方言适配 | ✓ 支持 | 通过自学习适配方言 |

**注意事项：**

- 自学习功能需要在安静环境下进行
- 建议每个命令词录制2-3次提高成功率

### CI-03T系列T2、T3、T4型号有什么区别？

**问题描述：**

需要了解CI-03T系列模块中T2、T3、T4等不同型号的区别，以便选择正确的型号进行开发。

**解决方案：**

**型号说明：**

目前市面上销售的CI-03T系列模块中，T2、T3、T4等后缀主要表示不同的生产批次或版本号，在**功能上没有差异**。

**关键点：**

1. **功能完全相同**
    - 所有CI-03T系列型号（包括CI-03T2、CI-03T3、CI-03T4）功能完全一致
    - 都使用CI1302主控芯片
    - 支持300条词条、双麦克风、自学习功能

2. **购买建议**
    - 选择当前有库存的型号即可
    - 不用过分纠结型号后缀的数字差异
    - 确认是CI-03T系列就可以

3. **开发兼容性**
    - 固件完全兼容，无需区分型号后缀
    - 开发工具和配置方法相同
    - 可以互换使用

**注意事项：**

- 购买时认准CI-03T系列即可，具体看哪个型号有现货
- 如遇到特殊后缀型号，可咨询客服确认功能差异
- 开发时按照CI-03T标准文档进行即可

**补充说明：**

- T2/T3/T4主要用于区分不同的SKU，功能与性能一致
- 选型时重点关注是否带晶振和麦克风数量：

    - 需要鼾声/哭声检测：选择带晶振版本（CI-03T3/CI-03T4）
    - 普通语音识别：带或不带晶振均可
    - 单麦：CI-03T1/CI-03T3；双麦：CI-03T2/CI-03T4（支持降噪和声源定位）
- 后缀数字不影响固件和功能选择，更多是库存管理标识

### CI1302和CI1303芯片的主要区别是什么？如何选择？

**问题描述：**

需要了解CI1302和CI1303芯片之间的差异，特别是在需要支持语音指令播放和背景音乐功能时的选型考虑。

**解决方案：**

**主要区别：**

CI1302和CI1303在功能上完全相同，唯一的区别是**内置Flash容量**：

- **CI1302**：2M字节 Flash
- **CI1303**：4M字节 Flash

**选型建议：**

1. **选择CI1302的场景**：

    - 只需要语音识别功能
    - 不需要存储音频文件
    - 成本敏感的项目

2. **选择CI1303的场景**：

    - 需要播放背景音乐
    - 需要存储大量音频提示音
    - 音乐文件较大，2M Flash不足够

**注意事项：**

- 两个芯片的封装完全相同，硬件设计兼容
- 开发费用相同，差异仅在芯片采购成本
- 可根据最终的音乐文件大小确定选型

### CI-03T的日志输出波特率是多少？

**问题描述：**

需要了解CI-03T模块的日志输出波特率，以便正确配置串口调试工具。

**解决方案：**

- CI-03T的日志输出波特率为**921600**（与烧录波特率相同）
- 调试时请将串口工具设置为921600
- 常用调试工具：Windows（SSCOM、串口助手）、Linux（minicom）、跨平台（PuTTY）

**注意事项：**

- 确保USB转串口驱动正确安装
- 核实串口号与波特率配置，避免乱码

### CI-03T支持CAN总线通信吗？

**问题描述：**

在车载等需要CAN总线的场景中使用CI-03T，询问是否支持CAN通信。

**解决方案：**

- CI-03T**不支持**直接的CAN总线通信（芯片外设无CAN控制器）
- 模块串口为TTL电平，如需接入CAN需外部转换

**CAN通信方案：**

1. **硬件转换**：TTL转RS485，再由RS485转CAN；或使用TTL转CAN转换模块
2. **软件配置**：在平台配置串口输出协议，并按照CAN格式封装数据

**注意事项：**

- 需要额外硬件转换电路，评估成本与实时性


- 可通过语音命令控制空调开关、温度、模式等

**空调控制的组合码说明：**

空调遥控器通常使用**组合码**进行控制，这是实现空调语音控制的重要概念：

1. **什么是组合码**：
    - 空调的每个操作（开关、温度、模式、风速）都需要一串红外编码
    - 例如：制冷26度模式下，同时包含模式=制冷、温度=26、风速=自动等信息
    - 这些信息组合成一个完整的红外码发送

2. **语音模块实现空调控制的方案**：
    - **方案一（仅语音模块）**：需要将所有可能的组合预先学习和存储
    - 例如：制冷模式 × 温度16-30度 × 风速3档 = 大量组合
    - 缺点：学习量大，存储空间占用多

    - **方案二（语音模块+码库）**：推荐方案
    - 使用带有红外码库的语音模块
    - 通过串口或内置码库动态生成组合码
    - 减少学习量，提高灵活性

    - **方案三（配合外部硬件）**：
    - 语音模块识别命令后，通过串口通知外部MCU
    - MCU使用专用空调红外芯片发送组合码
    - 适合复杂空调控制场景

**注意事项：**

- 使用红外功能需确保固件为新版本（支持红外收发）
- 旧版本固件需在平台重新生成才能获得红外功能
- 空调控制需要考虑组合码问题，不是简单的按键对应
- 开发时需测试空调品牌和型号的兼容性

### CI-03T模块带晶振和不带晶振版本如何选择？

**问题描述：**

在淘宝上看到CI-03T模块有带晶振和不带晶振两种版本，需要了解如何根据需求选择合适的版本。

**解决方案：**

**版本选择建议：**

1. **带晶振版**：

    - 适用于对时钟精度要求较高的场景
    - 在环境温差大的环境下表现更稳定
    - 晶振可以减少温度变化带来的漂移影响
    - 时钟精度更高，环境适应性更强

2. **不带晶振版**：

    - 适用于对时钟精度要求不高的普通应用场景
    - 成本相对较低
    - 适合温度稳定的使用环境
    - 基础语音识别功能不受影响

**选择依据：**

- **环境稳定性**：

    - 环境温度变化大：选择带晶振版
    - 室内恒温环境：可选择不带晶振版

- **精度要求**：

    - 需要精确时序控制：选择带晶振版
    - 仅基础语音控制：不带晶振版即可

- **成本考虑**：

    - 批量生产：带晶振版增加少量成本
    - 性价比要求：不带晶振版更具优势

- **特殊功能需求**：

    - 需要鼾声检测功能：**必须选择带晶振版**
    - 需要哭声检测功能：**必须选择带晶振版**
    - 普通语音指令控制：两个版本均可

**技术说明：**

- 两个版本的基础语音识别功能相同
- 主要差异在时钟精度和温度稳定性
- 不影响识别率和响应速度
- 封装和引脚完全兼容

**注意事项：**

- 根据实际应用场景选择
- 工业或户外应用建议带晶振版
- 家用产品可选不带晶振版
- 两个版本价格差异不大
- 鼾声/哭声检测等功能必须使用带晶振版本

### 如何配置CI-03T实现风扇转速的逐步控制和语音提示？

**问题描述：**

需要配置CI-03T模块实现通过语音指令控制风扇转速的逐步增加和减少，并在达到最大或最小转速时进行语音提示，同时确保风扇不会停转。

**解决方案：**

**实现方案：**

使用变量跟踪风扇转速状态，实现逐步控制：

1. **变量定义**：

    - 定义变量`flag`记录转速档位
    - 初始值设为1（最小转速）
    - 档位范围：1-5（五档调速）

2. **PWM配置**：
    ```
    Pin 9 → PWM0输出 → 控制风扇
    频率：2000Hz
    占空比范围：20%-100%
    ```

3. **转速映射**：
    ```
    档位1 → PWM占空比20%（最小转速，不停转）
    档位2 → PWM占空比40%
    档位3 → PWM占空比60%
    档位4 → PWM占空比80%
    档位5 → PWM占空比100%（最大转速）
    ```

**命令词配置：**

1. **打开风扇**：

    - 设置PWM占空比20%
    - 变量flag=1
    - 播报："风扇已打开"

2. **增加转速**：

    - 条件：flag<5
    - 动作：flag=flag+1，设置对应PWM占空比
    - 播报："转速已增加"（flag≤4时）
    - 播报："风速已最大啦主人"（flag=5时）

3. **减小转速**：

    - 条件：flag>1
    - 动作：flag=flag-1，设置对应PWM占空比
    - 播报："转速已减小"
    - 播报："风速已最小啦主人"（flag=1时）

4. **关闭风扇**：

    - 设置PWM占空比0
    - 可选择是否保留flag值

**配置要点：**

1. **PWM设置**：

    - 使用GPIO_A0(PWM)输出
    - 频率设置为2000Hz避免噪声
    - 最小占空比设为20%确保不停转

2. **变量逻辑**：

    - 每次增加转速变量+1
    - 变量≤4时播报"转速已增加"
    - 变量≥5时播报"风速已最大"

3. **安全保护**：

    - 最小转速保持20%占空比
    - 避免风扇完全停转
    - 最大转速限制在100%

**硬件连接：**

```
CI-03T Pin 9 → PWM输出 → 风扇驱动电路
风扇驱动电路 → DC风扇
独立电源 → 风扇供电
```

**调试技巧：**

- 先测试PWM输出是否正常
- 逐步验证每个档位的转速
- 测试边界条件（最大/最小）
- 验证语音提示功能

**注意事项：**

- PWM占空比不要设置为0（除非要关闭）
- 使用变量时注意初始值设置
- 建议添加开机初始化设置
- 风扇电源独立供电，不要使用模块电源

### 什么是自然说功能，哪些芯片支持？

**自然说功能说明：**

自然说是通过泛化词技术，让用户无需严格按照预设词条发音，系统可识别意思相近的多种表达方式，使交互更自然。

- **仅限命令词**：唤醒词不支持泛化
- **语言支持**：目前仅支持中文普通话

**支持型号：**

- **CI系列**：

    - CI-03T：支持自然说功能
    - CI-33T：支持自然说功能
    - CI-73T、CI-76Z：暂不支持，需要付费开发
- **Flash要求**：需要2M及以上容量
- **成本考虑**：批量价格约5元以上
- **开发服务**：不支持自然说的型号可付费开发，费用约2000元

**开发方式：**

1. 在开发平台选择"自然说"功能
2. 导入系统推荐词（如"打开"、"关闭"等）
3. 系统自动生成泛化词库
4. 可查看泛化词数量和效果

**注意事项：**

- 小语种（英语、阿拉伯语等）暂不支持自然说

- 带蓝牙功能的型号词条数较少，且不支持自然说

- **T1、T2、T3**等编号主要用于区分不同的生产批次或版本
    - 核心功能保持一致

**选型建议：**

- 工业环境或温差大的场合：选带晶振版本
- 普通家居环境：可选不带晶振版本
- 对稳定性要求高的项目：推荐带晶振版本

### 如何选择支持语音识别串口输出的模块？

**问题描述：**

需要选择一个支持语音识别并通过串口发送指令的模块，了解双麦克风模块的适用性和连接方式。

**解决方案：**

**推荐型号：**

- **CI-03T双麦模块**：支持语音识别+串口输出+声源定位
- **CI-73T/96Z系列**：仅支持语音识别+串口输出（低成本方案）
- **选型依据**：根据是否需要声源定位功能选择

![语音识别模块连接图](./img/3c335204.jpg)

**连接方式：**

1. **基础连接**
    - 麦克风：连接到模块的MIC输入
    - 串口输出：仅需连接UART RX引脚
    - 电源：连接5V或3.3V（根据模块规格）

2. **串口通信**
    - 波特率：默认9600bps（可配置）
    - 数据格式：8N1（8数据位，无校验，1停止位）
    - 输出内容：可自定义，支持帧头帧尾

**功能特性：**

- **命令词识别**：支持最多300条命令词
- **串口输出**：识别后自动发送预设指令
- **多语言支持**：中文和英文都可识别
- **英文识别**：需要使用英文固件制作

**使用示例：**

1. **平台配置**
    - 添加命令词列表
    - 设置对应的串口输出内容
    - 下载固件到模块

2. **硬件连接**
    ```
    MCU ---- RX ----> 模块TX
            GND ----> 模块GND
            5V ----> 模块VCC
    ```

**注意事项：**

- 只需识别功能时，单麦模块成本更低
- 双麦模块额外支持声源定位
- 串口输出内容可完全自定义
- 英文命令词需要用谐音或中文制作
