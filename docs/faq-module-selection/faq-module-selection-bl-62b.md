# BL-62B 模块选型 FAQ

本页用于整理 BL-62B 相关的模块选型问题。

### BL-62B与STM32F103C8T6有什么区别？

**问题描述：**

使用不同单片机型号时发现物料配置和功能存在差异，导致无法正常握手。

**解决方案：**

**核心区别：**

BL-62B和STM32F103C8T6是完全不同的平台，存在本质差异：

1. **主控芯片不同**

    - BL-62B：使用博流BL616芯片，WiFi+蓝牙双模芯片
    - STM32F103C8T6：使用意法STM32芯片，通用MCU

2. **平台架构差异**

    - BL-62B：基于RISC-V架构，专用于物联网应用
    - STM32F103C8T6：基于ARM Cortex-M3架构，通用微控制器

3. **功能定位不同**

    - BL-62B：主要用于无线连接、WiFi功能开发
    - STM32F103C8T6：通用控制、嵌入式应用开发

4. **开发环境差异**

    - BL-62B：需要使用博流专用开发工具和SDK
    - STM32F103C8T6：使用Keil、STM32CubeIDE等通用工具

**注意事项：**

- 两个平台不兼容，无法直接替换使用
- 开发资料和工具链完全不同
- 选型时需根据具体功能需求选择合适平台
- BL-62B不适合做通用的MCU控制应用

### 蓝牙模块能否与语音模块搭配使用？

**问题描述：**

需要确认蓝牙模块（如BL-62B）是否能与语音模块搭配使用，以及是否支持一主多从的连接模式。

**解决方案：**

**1. 模块搭配使用**

- 蓝牙模块可以与语音模块搭配使用
- 有专门的蓝牙+语音二合一模块可选
- 可根据项目需求选择合适方案

**2. 一主多从支持**

- 标准蓝牙协议为点对点连接
- 平台不支持一主多从模式配置
- 该功能需要SDK开发实现

**3. 实现方案**

- 方案一：选择蓝牙+语音二合一模块
- 方案二：使用主控MCU管理多个蓝牙模块
- 方案三：进行SDK开发定制一主多从功能

**注意事项：**

- 一主多从需要较复杂的协议处理
- 建议优先评估是否真的需要多从连接
- SDK开发需要较强的技术能力
