# SU-20T 烧录与调试 FAQ

本页用于整理 SU-20T 相关的烧录与调试问题。

### US513U6芯片烧录失败且更换芯片后仍无法解决怎么办？

**问题描述：**

使用US513U6芯片时无法完成烧录，尽管烧录进度条能跑完，但最终会报错。更换芯片后仍无法解决，需要明确烧录失败的原因及解决方案。

**解决方案：**

**问题根本原因：**

1. **芯片与模块不匹配**：

    - US513U6芯片实际对应SU-20T模块
    - 当前使用的是基于SU-22T（使用US513U61芯片）的固件
    - 芯片型号不匹配导致烧录失败

2. **硬件设计差异**：

    - SU-20T和SU-22T可以共板设计
    - 但DAC输出口也存在细微区别
    - 需要使用对应模块的资源

**解决步骤：**

1. **确认芯片型号**：

    - 检查芯片表面标识：US513U6（蜂鸟L）或US513U61（蜂鸟LPRO）
    - US513U6对应SU-20T，支持50条词条
    - US513U61对应SU-22T，支持61条词条

2. **使用正确资源**：

    - 下载SU-20T开发包而非SU-22T
    - 使用SU-20T的规格书和原理图
    - 烧录时选择SU-20T对应的配置

3. **固件配置调整**：

    - 在智能公元平台选择SU-20T模块
    - 配置不超过50条词条
    - 生成匹配的固件文件

**硬件兼容性说明：**

- SU-20T和SU-22T硬件兼容，可以共板设计
- 主要区别在于内部芯片型号和支持的词条数
- 如需要更多词条，建议使用US513U61（SU-22T）

**注意事项：**

- 采购时注意区分US513U6和US513U61
- 烧录工具需要选择正确的模块型号
- 保留好芯片包装便于后续识别
